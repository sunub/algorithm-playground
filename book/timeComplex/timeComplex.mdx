# 시간 복잡도를 사용하는 법

모든 컴퓨터의 성능이 동일하지는 않지만 대부분의 모든 컴퓨터들이 1초간 처리 가능한 계산 횟수는 `10^9 = 1,000,000,000` 회 정도입니다.

이러한 대략적인 시간을 알아야하는 이유는 무엇일까요? 알고리즘 문제를 풀다보면 모든 문제에는 **제약조건**이 붙습니다. 예를 들어 보겠습니다.

```
시간 제한 	메모리 제한
2 초	          512 MB

두 정수 N, M (3 ≤ N, M ≤ 10)
```

위와 같이 제약조건이 붙을 경우 어떤 알고리즘을 사용할 수 있는지에 대해서 생각할 수 있기 때문입니다. 위의 예시를 대상으로 생각하였을 경우 N이 3이상 10이하인 숫자로 작은 숫자에 해당하기 때문에 O(2^N)의 복잡도가 걸리는 알고리즘이라도 실행이 가능합니다. 하지만 N 값이 100,000이라고 생각하였을 경우 어느정도의 복잡도를 지닌 알고리즘을 실행할 수 있을까요?

```
N                  logN    NlogN            N^2     N^3    2^N     N!
100,000    17,       1,600,964     -           -           -            -
```

`-`는 10^9를 의미합니다. 10^9 이상의 시간이 걸리는 것은 우리가 상상할 수 없을 정도로 오랜 시간이 걸리기 때문에 무시해야 하기 때문에 자세히 표기하지 않았습니다. 위의 표를 예시로 100,000개의 숫자를 2초 내의 실행하기 위해서는 적어도 NlogN의 시간 복잡도를 가지는 알고리즘을 적용시켜 문제를 해결해야 한다는 힌트를 얻을 수 있기 때문에 시간 복잡도를 추측할 수 있는 것은 중요합니다.

O(N!)와 O(2^N) 과 같은 복잡도가 필요한 알고리즘은 지수 시간이 걸린다고 표현하고 반대로 복잡도의 상한이 제한된 경우 O(n^2), NlogN등은 다항 시간이 걸린다고 표현합니다. O(1) 시간이 걸리는 것들은 상수 시간이 걸린다고 표현합니다.
